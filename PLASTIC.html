Green Query Optimization

  • Taming Overheads with Plan Recycling.

The World of Query Optimization

  • A core, necessary database function.
  • Computationally intensive by its nature.
  • Goal is always optimal performance.

The Standard Process Today

  • Each query is optimized afresh.
  • This process has inherent overheads.
  • Plan reuse is extremely limited.
  • Requires close textual query resemblance.

The Core Inefficiency

  • Optimizing every single query is inefficient.
  • Overheads cannot be truly amortized.
  • Structurally similar queries are re-optimized.

The Downstream Consequences

  • Limits use of highest optimization levels.
  • Current plan reuse scope is narrow.
  • A missed opportunity for performance.

A New Way of Thinking

Recycle plans by clustering structurally similar queries, not just textually identical ones.

Introducing: PLASTIC

  • A new tool for plan recycling.
  • Groups queries into structural clusters.
  • Uses one plan for an entire cluster.
  • Dramatically reduces optimization overheads.

The PLASTIC Architecture

  • Feature Vector Extractor
  • Query Cluster Database
  • Similarity Check
  • Plan Generator
  • Optimizer invoked if no match.

Pillar 1: Amortize Costs

  • Significantly reduces total optimization time.
  • Cost is spread over many queries.
  • Frees up valuable system resources.

Pillar 2: Improve Efficiency

  • Optimizers can run at highest level.
  • This high cost is now affordable.
  • Wider scope for query matching.

Pillar 3: Adapt to Data

  • Assignment based on database statistics.
  • Plan choice adapts to current state.
  • System is dynamic and responsive.

Smart Feature: Variable Clustering

  • Matches volatility in the plan space.
  • Small clusters for high-volatility regions.
  • Large clusters for low-volatility regions.
  • Increases plan prediction accuracy.

Smart Feature: Decision Trees

  • New classifier for cluster identification.
  • Replaces expensive linear searching.
  • Makes cluster identification extremely fast.
